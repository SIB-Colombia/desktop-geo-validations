Desktop-geo-validations

Validations of geographical coordinates from files.

Download .zip Download .tar.gz View on GitHub

Introduction.

This app is util to check geospatial coordinates through txt files. For the moment this validation is only available for:

  • Counties
  • Marine Zones

Project main structure

The app has the following files:

  • client_config_areas.txt
  • client_config_municipios.txt
  • Validar.bat (Windows)
  • Validar.sh (Linus/Mac) and two folders:
  • AreasMaritimas
  • Municipios

Each folder contains information about:

  • input: There is the txt with the information to check.
  • output: There is a txt with the checked information.
  • shapefile: There are multiple files that contains the geographic information.
  • SHPMunicipioV.jar or SHPAreaMaritimaV.jar

Note

You can download:

  • Shapefile: https://s3.amazonaws.com/IPT/shapes/LimitesMaritimos.zip and https://s3.amazonaws.com/IPT/shapes/Departamentos(2012).zip
  • Java ARchive: https://github.com/SIB-Colombia/desktop-geo-validations/tree/gh-pages/Java%20ARchive

Structure of txt files

Input txt

For Marine Zone case:

The txt need to have the name "areaMaritimas.txt" and the following information in the same order:

  • id
  • latitude
  • longitude
  • Marine Zone name

For Counties case:

The txt needs to have the name "municipios.txt" and the following information in the same order:

  • id
  • latitude
  • longitude
  • County name
  • Department name

Output txt

For Marine Zone case:

The txt shows the information in the following order:

  • id
  • latitude
  • longitude
  • Marine Zone name
  • Marine Zone name calculated
  • True or False (marine zone check)

For Counties case:

The txt shows the information in the following order:

  • id
  • latitude
  • longitude
  • Department name
  • Department name calculated
  • True or False (department check)
  • County name
  • County name calculated
  • True or False (county check)
